# Bonnes pratiques de support infrastructure

## Rester factuel

Décrire ce qui est observé : message d'erreur, heure, poste concerné, utilisateur, application, fréquence du problème. Éviter les suppositions non vérifiées.

## Commencer par le périmètre

Avant de chercher une cause complexe, identifier si l'incident touche :

- un seul utilisateur ;
- un seul poste ;
- une équipe ;
- un atelier ;
- plusieurs sites ;
- une application précise.

## Vérifier les bases

Les contrôles simples résolvent ou orientent beaucoup d'incidents :

- alimentation électrique ;
- câble réseau ou Wi-Fi ;
- redémarrage si possible ;
- espace disque ;
- compte verrouillé ou mot de passe expiré ;
- imprimante allumée et consommables présents ;
- accès à d'autres sites ou applications.

## Ne pas interrompre la production sans raison

Avant de redémarrer un poste, une imprimante ou un équipement utilisé en production, vérifier si l'action peut gêner une opération en cours. Si besoin, demander l'accord du responsable ou suivre la procédure interne.

## Documenter les actions

Un ticket utile contient :

- le contexte ;
- les symptômes ;
- les personnes ou postes impactés ;
- les tests réalisés ;
- les résultats obtenus ;
- la solution appliquée ou la raison de l'escalade.

## Respecter les comptes et les droits

Ne pas prêter son compte, ne pas utiliser le compte d'un collègue et ne pas donner de droits sans validation. Les accès doivent rester traçables.

## Communiquer simplement

Informer l'utilisateur avec des phrases claires :

- ce qui est constaté ;
- ce qui est en cours de vérification ;
- ce qui est résolu ;
- ce qui est transmis à un autre niveau si nécessaire.

## Escalader proprement

Escalader quand le problème dépasse le niveau autorisé ou les compétences disponibles. Fournir toutes les informations collectées pour éviter de refaire les mêmes tests.

## Garder une approche progressive

Traiter d'abord les causes simples et probables, puis avancer vers les causes plus techniques. Une bonne méthode évite de perdre du temps et limite les erreurs.
